    第8章 01： 四个和弦，一千首歌#

    拿起一把吉他。学 G、C、D 和 E minor。就这四个和弦。练到切换顺滑——大概每天二十分钟，练一周。

    然后打开歌本。你能弹《Let It Be》。能弹《No Woman, No Cry》。能弹《With or Without You》。能弹《Someone Like You》。能弹《Country Roads》。还能弹几百首。

    四个和弦。几百首歌。不需要音乐学位。

    这不是取巧。这是技能运作方式中的一个基本规律。一旦看见了，就再也看不回去了。

    技能的幂律#

    大多数人学新技能的时候，把所有元素当成同等重要的。把时间和注意力均匀撒在整个版图上。感觉很全面。感觉很负责。但效率极低。

    原因在这里：几乎所有领域，极少数元素出现在绝大多数的使用场景中。这不是巧合——这是一个叫幂律分布（Power-Law Distribution）的数学规律。少数东西极其重要。多数东西几乎无关紧要。两者之间的差距不是一点点，是巨大的。

    音乐里，四个和弦进行就能覆盖大部分流行歌曲。做饭中，五种技法——煎、烤、煮、炖、炒——能应付大部分家常菜。编程中，几个核心概念——变量、循环、条件判断、函数、基础数据结构——出现在几乎每一个程序里。语言学习中，最常用的 300 个词大约占日常对话的 65%。

    少数核心元素覆盖了大部分实际应用。找到它们，是早期学习中杠杆最高的一步。

    如果你先找到并集中精力在这少数元素上，你获得的能力是不成比例的。不是专业水平。不是精通。但是可用的、实际的、能带来乐趣的能力——比均匀分配的学法快得多。

    找到核心元素#

    怎么在任何领域里找到核心元素？

    比你想的简单。看频率。

    不是看重要性。不是看难度。不是看含金量。看频率。什么东西在实际使用中反复出现？从业者最常做什么？什么出现在大多数真实场景里？

    这跟"我应该先学什么"是不同的问题——那个问题通常指向教科书式的、为全面理解而设计的课程顺序。频率分析指向的是以练习为导向、为快速获得能力而设计的学习顺序。

    高频元素清单#

    怎么建你的清单：

    第一步：找十个这项技能在实际使用中的例子。 不是教程。不是理论。是实际使用。十首你想弹的歌。十道你想做的菜。十个做了有用事情的程序。十段你在学的语言的对话。

    第二步：找出反复出现的元素。 横向看你的十个例子。什么出现在大多数里？哪些和弦出现在十首歌中的七首？哪些食材出现在十道菜中的八道？哪些编程概念在十个程序里都有？

    第三步：按频率排序。 从最高频到最低频排列。清单顶端——出现在最多例子中的元素——就是你的核心元素。

    第四步：先学顶端的元素。 前五到十个以下的，暂时忽略。以后再说。现在只专注于给你最广覆盖面的那些元素。

    不是走捷径。是优化排序。你不是在跳过内容——你是在按影响力排列它们。

    覆盖率测试#

    有了核心元素之后，测试一下。问：“只靠这些元素，我能应付多大比例的目标场景？”

    吉他：四个和弦大约覆盖 70-80% 的流行歌曲。极少的学习投入换来极大的回报。

    做饭：五种基本技法大约覆盖 90% 的家常菜需求。你不需要会低温慢煮、巧克力调温或从零熬高汤，就能天天自己做饭吃一年。

    日常语言：300 个词覆盖大约 65% 的日常对话。再加 200 个就到 80%。你不需要 5000 个词就能进行有用的对话。

    这就是覆盖率测试：核心元素覆盖的场景数除以总场景数。超过 60%？你找到了最小可行入口（Minimum Viable Entry）。你可以开始在这个领域里操作了——不完美，不全面，但能用。

    四个和弦能弹几百首歌。不需要音乐学位。这就是幂律在为你工作。

    “够用"不是贬义词#

    有一种对这个方法的抵触，我想正面回应。它听起来像这样：“但如果你只会四个和弦，你又不是真正的音乐人。”

    没错。但也不相干。

    这里的目标不是让你成为音乐人。是让你成为一个会弹音乐的人。这两者之间差别巨大。

    音乐人花了多年钻研理论、打磨技术、发展艺术表达、进行专业演出。了不起的目标。需要几千小时。

    一个会弹音乐的人呢？就是一个能在篝火旁弹几首大家都知道的歌的人。聚会上拿起吉他弹弹的人。下班后拨弄几首熟悉旋律的人。从一件乐器中获得真实快乐的人。

    第二种人——四个和弦，一百首歌——拥有的东西是真实的。实用的。令人愉悦的。而且他们用二十个小时就到了那里，不是二十年。

    问题不是"这算精通吗？“而是"这有用而且让人开心吗？“如果是，你就跨过了能力门槛。以后想深入随时可以。但你不必深入也已经拥有了有价值的东西。

    能力光谱#

    把能力想成一个光谱，不是一个二选一：

    0% --------[门槛]------------ 100%
不会         能用地做到       世界级

    大多数人以为必须接近右端才算数。他们看到到 100% 的距离，觉得太远了。于是根本不开始。

    门槛系统说的是：到门槛那个标记就够了。够获得价值。够获得乐趣。够决定你想不想继续往深走。

    门槛不在 50%。对大多数实用技能来说，它在完整能力的 20-30% 左右。四个和弦大概只占音乐理论的 5%。但它覆盖了你真正想弹的歌的 70-80%。幂律——一小片知识覆盖一大片应用。

    Darius 和厨房#

    Darius 一直说自己不会做饭。一周五天叫外卖。厨房里有一个咖啡机，一个微波炉，和灰尘。

    一个朋友挑战他：一个月内学会做五道菜。不要精致的。不要能发 Instagram 的。五道能在工作日晚上做的简单菜。

    Darius 从找核心元素开始。他看了二十个简单菜谱，记下反复出现的东西。煎炒出现在十六个里。洋葱、大蒜、橄榄油、盐、胡椒出现在十八个里。蛋白质加蔬菜加主食的结构出现在全部二十个里。

    他专注在这些上面。学会了不把洋葱和蒜炒糊。学会了一边做一边调味、经常尝。学会了煮饭、煮面、烤蔬菜。反复练这些基本功。

    第二周，他能用冰箱里的任何东西炒个菜了。第三周，会做带酱汁的意面。第四周，他有了五道菜——而且开始即兴发挥了。往里加菜谱上没有的食材。靠嘴巴调味。把几样东西的时间掐到同时出锅。

    他是厨师吗？不是。他能大部分晚上不叫外卖给自己做一顿像样的晚饭吗？能。而那种满足感——用自己的手把生的食材变成一顿饭——超出了他的预期。

    Darius 不是学了 5% 的烹饪。他学的是覆盖了他 80% 需求的那 5%。幂律。最小可行入口。

    从"我不会做饭"到"我大部分晚上自己做饭”，三十天。不是天赋。是瞄准了对的那 5%。

    投入产出比最大化#

    这个方法从根本上讲就是投入产出比。不是财务上的——是每小时练习换来的能力回报。

    把学习时间均匀分散到所有元素上，能力线性增长。每小时加差不多等量的能力。缓慢而稳定。

    集中在高频核心元素上，早期的能力增长是指数级的。每小时花在核心元素上的回报不成比例地大，因为那个元素出现在太多使用场景里。

    这不是永久策略。最终你会用尽核心元素，需要转向频率更低的。回报会恢复正常。但在关键的头二十个小时——大多数人要么决定坚持一项技能、要么放弃它的那个窗口——这种集中策略产出的可用能力远远超出均匀分配。

    投入产出比计算#

    一个简单的思考方式：

    • 学和弦 G：出现在你想弹的流行歌曲的约 40% 中
    • 学和弦 C：约 35%
    • 学和弦 D：约 30%
    • 学和弦 Em：约 25%
    • 学增六和弦：约 0.1%

    前四个和弦中的每一个，都以极小的投入换来巨大的覆盖面。增六和弦在音乐理论上同样成立——但对你当前的目标来说，实际回报接近于零。

    不是说它没用。爵士乐手或古典作曲家需要它。但对一个正在跨越吉他能力门槛的人来说，它是噪音。在核心四个和弦之前学它，是对早期学习中最珍贵资源的错配：时间。

    乐趣就是燃料#

    这个等式里还有一个要素——也许是最重要的。

    当你专注于高频核心元素时，你很快就能达到可用的能力水平。而可用的能力是有趣的。弹一首别人认得出的歌是有趣的。做一顿好吃的饭是有趣的。用新语言进行对话——哪怕很基础——是有趣的。写一个真的能跑的程序是有趣的。

    乐趣之所以重要，因为乐趣就是持续性。一件事情做起来有乐趣的时候，你不需要意志力来坚持。不需要纪律来出现。不需要动力技巧或监督伙伴。你练习因为你想练。因为感觉好。因为这个活动本身已经成了奖励。

    良性循环：

    1. 聚焦核心元素 → 快速达到可用能力
    2. 可用能力 → 乐趣和满足感
    3. 乐趣和满足感 → 不需要意志力就能继续练习
    4. 持续练习 → 能力随时间深化
    5. 更深的能力 → 更多乐趣，更多满足感

    跟全面学习法对比：

    1. 试图什么都学 → 所有元素上都进展缓慢
    2. 进展缓慢 → 几周甚至几个月都没有可用的能力
    3. 没有可用能力 → 没有乐趣，只有努力
    4. 没有乐趣 → 需要意志力来坚持
    5. 意志力耗尽 → 放弃这项技能

    当学习本身变得有趣，坚持就不再需要意志力了。通往乐趣的最快路径，穿过核心元素。

    大多数放弃一项技能的人不是缺乏自律。他们缺的是早期的胜利。缺的是"我居然真的能做到"的感觉。幂律方法快速送达这种感觉——不是靠降低标准，而是靠瞄准每小时产出最多能力的那些元素。

    你的核心元素冲刺#

    行动计划：

    这周： 选你想学的技能。找十个这项技能在实际中使用的例子。找出出现频率最高的五到十个元素。写下来。这就是你的核心元素清单。

    下周： 只练这些元素。其他什么都不管。不看高级技巧的教程。不探索边缘情况。就核心元素，练到顺滑。

    第三周： 把你的核心元素应用到一个真实场景中。弹一首歌。做一顿饭。写一个程序。进行一段对话。看看它们能覆盖多少。

    第四周及以后： 加入下一层级的元素——那些出现在十个例子中的五个而不是八个里的。你的能力在扩展，但最难的部分已经完成了。你已经跨过了门槛。

    找到你技能的那四个和弦。练到手指不用想就能动。然后打开歌本，看看它们能带你走多远。答案会让你惊讶。
